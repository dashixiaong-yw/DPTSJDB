# 对比结果详情页重新设计 — 三版方案

## 现状分析

当前结果页 `src/app/result/[taskId]/page.tsx`（386行）存在以下问题：

1. **信息密度低**：每个店铺/行都展开一个完整表格，大量"一致"项占据视觉空间，用户难以快速定位问题
2. **缺乏筛选**：无法只看"不一致"或"缺失"项，必须逐行扫描
3. **统计展示单调**：4个数字卡片+进度条，缺乏直观的分布可视化
4. **无汇总视图**：没有按店铺/月份汇总的快速概览
5. **移动端不友好**：宽表格在手机上需要横向滚动

## 灵感来源

综合研究了以下产品的设计模式：
- **Diffchecker / GitHub PR Diff**：差异高亮、并排对比、问题项优先
- **Stripe Dashboard**：极简统计卡片 + 筛选标签 + 分层展示
- **Linear**：列表式问题追踪，状态标签清晰，信息密度高
- **同花顺/财务对账系统**：饼状图/柱状图可视化、汇总表格
- **Vercel Dashboard**：黑白极简、Geist 字体、高信息密度

## 方案 A：手风琴折叠式（问题优先）

**灵感**：Linear 的问题列表 + GitHub Diff 的高亮模式

**核心理念**：问题项默认展开置顶，一致项折叠，减少视觉噪音

```
┌─────────────────────────────────────────────┐
│ ← 返回首页                                  │
│ 比对结果                                    │
│ 文件: xxx.xlsx | 平台: 淘宝                 │
├─────────────────────────────────────────────┤
│ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐        │
│ │ 总计 │ │ 一致 │ │不一致│ │ 缺失 │        │  ← 统计卡片
│ │  120 │ │  98  │ │  15  │ │   7  │        │     带环形图
│ │  ◔   │ │  ◔   │ │  ◔   │ │  ◔   │        │
│ └──────┘ └──────┘ └──────┘ └──────┘        │
│ ████████████████████░░░  81.7%              │  ← 匹配率
├─────────────────────────────────────────────┤
│ [全部] [不一致(15)] [缺失(7)]  🔍搜索字段   │  ← 筛选标签
├─────────────────────────────────────────────┤
│ ▼ 店铺A | 3月 | 5项 · 2不一致              │  ← 手风琴：有问题默认展开
│   ┌─────────────────────────────────────┐   │
│   │ 字段    │ 表格值  │ OCR值  │ 状态  │   │
│   │ 广告费  │ 1234.5  │ 1230   │  ✗    │   │  ← 仅展示问题项
│   │ 佣金    │ -       │ 56.7   │  ⚠    │   │
│   └─────────────────────────────────────┘   │
│   [展开全部 5 项 ▼]                          │
│                                             │
│ ▶ 店铺B | 3月 | 3项 · 全部一致 ✓           │  ← 全部一致默认折叠
│ ▼ 店铺C | 4月 | 4项 · 1缺失                │
│   ...                                       │
├─────────────────────────────────────────────┤
│ [下载标记文件] [导出JSON报告]                │
└─────────────────────────────────────────────┘
```

**优点**：聚焦问题、空间高效、筛选直观
**缺点**：需要多次展开才能看全貌

## 方案 B：双栏 Diff 对比式

**灵感**：GitHub PR 的 side-by-side diff 视图 + Diffchecker

**核心理念**：表格值和 OCR 值左右并排，差异高亮标红，类似代码 diff

```
┌─────────────────────────────────────────────┐
│ ← 返回首页                                  │
│ 比对结果                                    │
│ 文件: xxx.xlsx | 平台: 淘宝                 │
├─────────────────────────────────────────────┤
│ ████████████████████░░░  81.7%              │
│ 120项 · 98一致 · 15不一致 · 7缺失           │  ← 紧凑统计行
├─────────────────────────────────────────────┤
│ [全部] [不一致] [缺失]  🔍  店铺A ▼ 月份▼  │  ← 筛选 + 下拉筛选
├─────────────────────────────────────────────┤
│          表格值          │       OCR值       │  ← 双栏对比
│─────────────────────────┼──────────────────│
│ 店铺A | 3月             │                   │
│  广告费   1234.5        │  广告费   1230    │  ← 差异行红底
│  佣金     890.00        │  佣金     890.00  │  ← 一致行绿底
│  退款     -             │  退款     56.70   │  ← 缺失行灰底
│                         │                   │
│ 店铺B | 3月             │                   │
│  ...                    │  ...              │
├─────────────────────────────────────────────┤
│ [下载标记文件] [导出JSON报告]                │
└─────────────────────────────────────────────┘
```

**优点**：差异一目了然、对比直观、专业感强
**缺点**：移动端双栏体验差（需降级为单栏）、信息密度高可能视觉疲劳

## 方案 C：卡片流式（移动优先）

**灵感**：Stripe Dashboard 的交易列表 + 同花顺的卡片式数据展示

**核心理念**：每个比对项一张卡片，问题项置顶，状态标签醒目，移动端原生体验

```
┌─────────────────────────────────────────────┐

│ 比对结果                                    │
│ 文件: xxx.xlsx | 平台: 淘宝                 │
├─────────────────────────────────────────────┤
│ ┌───────────────────────────────────────┐   │
│ │  ████████████████████░░░  81.7%       │   │
│ │  120项 · 98一致 · 15不一致 · 7缺失    │   │  ← 大号匹配率卡片
│ └───────────────────────────────────────┘   │
├─────────────────────────────────────────────┤
│ [全部] [不一致] [缺失]                       │  ← 筛选标签
├─────────────────────────────────────────────┤
│ ┌─ 店铺A · 3月 ──────────── 2不一致 ──┐   │
│ │                                      │   │
│ │  广告费                    ✗ 不一致   │   │  ← 问题卡片
│ │  表格: 1234.5                        │   │
│ │  OCR:  1230                          │   │
│ │  差异: -4.5                          │   │
│ │──────────────────────────────────────│   │
│ │  佣金                      ⚠ 缺失    │   │  ← 缺失卡片
│ │  表格: -                             │   │
│ │  OCR:  56.7                          │   │

│                                             │
│ ┌─ 店铺B · 3月 ──────────── ✓ 全部一致 ─┐ │
│ │  3项全部一致，点击展开详情              │ │  ← 一致项折叠为单行
│ └──────────────────────────────────────┘   │
│                                             │
│ ┌─ 店铺C · 4月 ──────────── 1缺失 ─────┐ │
│ │  ...                                  │ │
│ └──────────────────────────────────────┘   │
├─────────────────────────────────────────────┤
```

**优点**：移动端体验最佳、问题项差异值清晰、视觉层次分明
**缺点**：桌面端信息密度偏低、滚动距离长

## 三版方案对比

| 维度 | A 手风琴折叠 | B 双栏 Diff | C 卡片流 |
|------|:----------:|:----------:|:-------:|
| 问题定位速度 | ★★★★ | ★★★★★ | ★★★ |
| 信息密度 | ★★★★ | ★★★★★ | ★★ |
| 移动端体验 | ★★★ | ★★ | ★★★★★ |
| 实现复杂度 | 中 | 高 | 低 |
| 专业感 | ★★★ | ★★★★★ | ★★★ |
| 数据量大时 | ★★★★ | ★★★ | ★★ |

## 推荐：方案 A（手风琴折叠式）

理由：
1. 平衡了信息密度和问题定位速度
2. 实现复杂度适中，不引入额外依赖
3. 手风琴模式在桌面和移动端都有良好体验
4. 融合了方案 B 的差异高亮和方案 C 的移动端适配思路

最终实现会在方案 A 基础上吸收：
- 方案 B 的差异高亮（不一致项表格值 vs OCR值并排标红）
- 方案 C 的移动端适配（小屏下表格变卡片）

## 实现要点

1. 使用已有 shadcn/ui 组件：`Accordion`、`Tabs`、`Badge`、`Button`、`Card`、`Input`
2. 纯 CSS 环形图（conic-gradient），不引入额外 chart 库
3. 筛选逻辑用 useState 管理，纯前端过滤
4. 保持现有数据接口 `ResultData` 不变
5. 保留下载标记文件和导出JSON功能
6. 移动端：768px 以下表格自动切换为卡片布局
