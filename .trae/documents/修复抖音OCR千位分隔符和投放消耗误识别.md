# 修复抖音OCR千位分隔符和投放消耗误识别

## 摘要

修复抖音月度账单截图的两个OCR识别错误：
1. 退款金额 `7003.6` 被识别为 `7033.6`（单数字位视觉误读）
2. 投放消耗 `0` 被识别为 `1372.36`（字段匹配关键词过于宽泛）

## 当前状态分析

### 问题1：单数字位视觉误读

**错误对比**：`7,003.60` → `7033.6`
- 正确：`7` `0` `0` `3` `.` `6` `0` → `7003.60`
- 错误：`7` `0` `3` `3` `.` `6` → `7033.6`

**根本原因**：模型把第二个 `0` 误读成了 `3`，这是单数字位的视觉识别错误。

**当前提示词问题**：[ocr-service.ts](file:///d:/trea项目/多平台账单对比系统/src/lib/ocr-service.ts) 的 `buildDouyinShopMonthlyPrompt`（L570-642）缺少：
1. 千位分隔符的详细说明（只有一行"去掉逗号"）
2. 金额格式的正确/错误示例
3. 逐位确认的要求

对比 `buildDouyinExpensePrompt`（L675-691），已有完整的千位分隔符说明（包含正确/错误示例）。

### 问题2：投放消耗关键词过于宽泛

```typescript
// 第599-602行：投放消耗的识别规则
3. **投放消耗**
   - 查找包含"投放消耗"或"投放"或"消耗"的卡片
   - 提取其数值
   - 如果没有此卡片，设为0
```

**根本原因**：页面中"收支概况"区域显示"支出金额 ¥1,372.36"，模型误将"支出"相关的值匹配到了"投放消耗"字段（关键词"消耗"过于宽泛）。

## 修改方案

### 修改文件：`src/lib/ocr-service.ts`

**改动1：增强 `buildDouyinShopMonthlyPrompt` 的千位分隔符说明**

在关键规则部分（第637行附近）添加与 `buildDouyinExpensePrompt` 相同级别的千位分隔符说明：
- 添加正确示例（`¥7,003.60` → `7003.60`）
- 添加错误示例（必须避免的错误）
- 添加逐位确认要求

**改动2：收紧投放消耗的关键词匹配规则**

将投放消耗的匹配规则从"投放消耗"或"投放"或"消耗"改为只匹配"投放消耗"或"投放"，移除单独的"消耗"关键词。

**改动3：添加字段区分规则**

明确说明"支出金额"和"投放消耗"是不同的字段，投放消耗必须包含"投放"二字，避免混淆。
